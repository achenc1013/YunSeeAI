## 💡 智能特性

### 1. 上下文感知

第一次查询时指定URL，后续可以省略：

```
你: http://192.168.20.144/ 用了什么CMS？
AI: [识别CMS] → WordPress

你: 开通了哪些服务？  ← 无需重复URL
AI: [自动使用上次目标进行端口扫描]

你: 端口扫描一下  ← 无需重复URL
AI: [继续使用同一目标]
```

### 2. 自然语言理解

AI理解各种自然表达方式：

| 用户说法 | AI理解为 |
|---------|---------|
| "开通了哪些服务" | 端口扫描 |
| "提供了什么服务" | 端口扫描 |
| "运行了什么服务" | 端口扫描 |
| "用了什么CMS" | CMS识别 |
| "什么框架" | 框架识别 |
| "有什么漏洞" | 漏洞扫描 |

### 3. URL格式灵活

支持多种URL格式：

```
✅ http://example.com
✅ https://example.com
✅ example.com
✅ http://192.168.1.1
✅ http://192.168.1.1:8080
✅ http://example.com:3000/path
```

## 🚫 常见误区

### ❌ 错误示例

```
❌ "扫描"（没有指定目标，也没有历史目标）
❌ "这个网站"（需要明确URL）
❌ "刚才那个"（AI无法理解模糊指代）
```

### ✅ 正确示例

```
✅ "扫描 http://example.com"
✅ "http://example.com 扫描一下"
✅ 先查询一次带URL的，后续可以省略
```

## 🎯 最佳实践

### 1. 首次查询要完整

```
✅ 第一次提到新目标时，提供完整URL
示例: "http://192.168.20.144/ 用了什么CMS？"
```

### 2. 后续可以简化

```
✅ 对同一目标的后续查询可以省略URL
示例: "端口扫描一下"、"有什么漏洞？"
```

### 3. 切换目标时要明确

```
✅ 切换到新目标时，提供新的完整URL
示例: "http://example.com 用了什么框架？"
```
